# 📝 內容紀錄

# 1. 安裝WSL
打開Power shell，輸入指令來安裝WSL  
`wsl --install -d Ubuntu-24.04`  
安裝完成後，重新開機  

![](pics/Pasted%20image%2020260404234938.png)

電腦重開後可能會自動跳出帳號創建的畫面  
可以接著輸入自己的帳號名稱與密碼  
未來可以在這個帳號創建專屬的環境  
但不影響後續Truchas-Lab的安裝 
我們主要是為了能使用wsl的相關指令
![](pics/Pasted%20image%2020260405000204.png)

# 2. 把.7z檔解壓縮成.tar檔放到D槽
右鍵解壓縮 Truchas_Lab_116_Parallel_20260408-Final_2.7z  
得到壓縮檔 Truchas_Lab_116_Parallel_20260408-Final.tar  
把.tar檔拉到D槽，或是記下.tar檔的路徑來修正步驟3.的指令    

# 3. 在D槽建立資料夾WSL_Truchas
打開Power Shell 輸入 以下指令來在D槽建立資料夾WSL_Truchas
`mkdir D:\WSL_Truchas`  


接著在Power Shell 輸入以下指令來匯入打包好的WSL環境 (指令間隔是空格不是換行)
如果你剛剛有把 tar檔 放到D槽，指令就會是

```
wsl --import Truchas-Lab D:\WSL_Truchas D:\Truchas_Lab_116_Parallel_20260408-Final.tar
```

意思是：  
  
	wsl 啟用 WSL功能  
	--import 使用匯入功能  
	Truchas-Lab 匯入後將WSL環境取名  
	D:\WSL_Truchas 匯入的目的地  
	D:\Truchas_Lab_116_Parallel_20260408-Final.tar 匯入的檔案位置  

如果出現錯誤訊息，請確認你的.tar檔是否拉到D槽了，不然就依其所在位置更新指令。    
或是你很不幸的已經安裝了舊版本，可以參照這份文件最底下的[安裝教學_20260408](安裝教學_20260408.md#問題集)--砍掉重練。

安裝好後，輸入`wsl -d Truchas-Lab`來登入預設環境 Truchas-Lab    
登入界面會顯示系統狀態以及使用說明    
![](pics/Pasted%20image%2020260408032744.png)

![](pics/Pasted%20image%2020260408034752.png)
![](pics/Pasted%20image%2020260408034806.png)

可以將登入指令自訂成其他指令  
在Power Shell 輸入 `notepad $PROFILE` 來開啟記事本 (此時須未登入WSL)  
在最後一行加入  
`function ABCD { wsl -d Truchas-Lab }`  
以後在Power Shell輸入`ABCD`即可登入Truchas-Lab  
  
# 4. 相關指令
基本上你會用到的指令都寫在登入界面了，想看的話可以重開一個分頁登入

登入WSL後  

	run_compile：自動編譯  
	run_sim : 自動跳轉到116.../problems/tests+跑模擬   
	gmv : 開啟GMV軟體  
	pwd_win : 顯示當前windows路徑  
	open_win : 在windows資料夾打開  
	exit : 普通退出（可能會有背景程式繼續跑）  
	bye : 安全退出（全部停止才退出）  
  
退出WSL後  
  
	wsl --shutdown 徹底結束WSL  

# 問題集

## - 砍掉重練： 萬憂解  
	1.  徹底關閉所有 WSL 程序  
	wsl --shutdown  
	
	2. 列出並註銷（刪除）那個錯誤的 Ubuntu   
	wsl -l -v  
	wsl --unregister Ubuntu  
	
	3. 註銷（刪除）之前匯入失敗的 Truchas-Lab  
	wsl --unregister Truchas-Lab  
	
	4. 確認清單已空 (應該只剩 "沒有安裝的散發套件")  
	wsl -l -v  
	
	5. 建立乾淨的存放路徑  
	mkdir D:\WSL_Truchas -ErrorAction SilentlyContinue  
	
	6. 重新匯入 Truchas-Lab  
	見安裝步驟 4.  
  
## - 快速修復步驟（解決Power Shell卡頓）

	A. 可能是防毒的問題，關掉試試看。
	
	B. 我們直接把那個「作怪」的設定檔刪除或重置，讓 PowerShell 回歸純淨狀態。請在 PowerShell 視窗中依序輸入以下步驟：
	
	#### 1. 找出到底是哪個檔案在卡
	
	輸入這行（如果這行也卡住，請直接看步驟 2）：
	
	PowerShell
	
	```
	test-path $PROFILE
	```
	
	#### 2. 強制刪除設定檔 (砍掉重練)
	
	不管是因為 `ABCD` 寫錯還是路徑問題，先刪掉它最快：
	
	PowerShell
	
	```
	Remove-Item $PROFILE -Force
	```
	
	#### 3. 測試是否恢復正常
	
	關閉 PowerShell 視窗，重新開啟一個。如果它能在 **1 秒內** 出現提示字元，代表修復成功。  
   
  
  ## - 搬運檔案

	經測試， Win 11桌電可以透過以下方式在個人電腦與容器間搬運檔案：
		1. Xftp 
		2. windows 檔案總管（即平常開資料夾的方式）
		3. Linux 指令
	之後在相同位置會產生後綴為 Zone.Identifier 的同名檔案，此類檔案可自行刪除。
	
	然而，有時候會遇到搬運inp檔後無法正常模擬的情況，可能是Windows跟Linux在換行格式、檔案權限上有所衝突，可以嘗試替換~/.bashrc中的fix函數為下方版本。
	
	在登入WSL後，輸入`nano ~/.bashrc`開始編輯.bashrc檔案，或是輸入`open_win`在Windows界面編輯.bashrc檔案，完成後重開WSL或是輸入`source ~/.bashrc`來使設定生效。之後cd到需要修正的檔案路徑後，輸入`fix 檔案名稱`來修正。
	
	
```
# --- 終極版快速修復函數 (WSL/Windows 檔案救援) ---
fix() {
    local target_file="$1"

    if [ -z "$target_file" ]; then
        echo "❌ 使用方法: fix [檔案名稱]"
        return 1
    fi

    if [ ! -e "$target_file" ]; then
        echo "❌ 錯誤：找不到檔案 '$target_file'"
        return 1
    fi

    echo "⚙️ 正在執行深度修復：$target_file ..."

    # 1. 強制轉為 Linux 換行格式 (移除 Windows 的 \r)
    dos2unix "$target_file" 2>/dev/null

    # 2. 確保權限全開 (777：讀、寫、執行完全不設防)
    chmod 777 "$target_file"

    # 3. 確保擁有者回歸 WSL 使用者 (避免 root 或 Windows 帳號鎖定)
    # 使用 sudo 確保強制執行
    sudo chown userslider60213:userslider60213 "$target_file"

    echo "✅ [完成] 換行格式、777 權限、擁有者已全數校正。"
}
```
